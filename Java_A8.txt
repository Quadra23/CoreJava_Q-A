Q1.What is ORM in Hibernate?
Ans- ORM stands for Object-Relational Mapping. In the context of Hibernate, ORM refers to the technique of mapping Java objects to relational database tables, and vice versa, automatically. It eliminates the need for manual SQL coding and simplifies the interaction between Java objects and databases by providing a layer of abstraction.

Q2.What are the advantages of Hibernate over JDBC?
Ans-Advantages of Hibernate over JDBC include:

Simplified Database Access: Hibernate handles the mapping between Java objects and database tables, eliminating the need for manual SQL coding and reducing development effort.

Object-Oriented Approach: Hibernate allows developers to work with objects directly, providing a more natural and intuitive way to interact with databases.

Increased Productivity: Hibernate automates repetitive tasks such as connection management, transaction handling, and SQL generation, saving development time and effort.

Database Independence: Hibernate abstracts the underlying database details, making it easier to switch between different databases without changing the application code.

Caching and Performance Optimization: Hibernate provides caching mechanisms that can significantly improve application performance by reducing database access and query execution.

Improved Maintainability: The use of Hibernate simplifies code maintenance, as it provides a higher level of abstraction and makes the codebase more readable and maintainable.

Support for Object Relationships: Hibernate supports mapping complex object relationships such as one-to-one, one-to-many, and many-to-many, simplifying the management of object associations.

Q3.What are some of the important interfaces of Hibernate framework?
Ans-Some important interfaces of the Hibernate framework include:

SessionFactory: Represents a factory for creating Session instances, which are used to interact with the database.

Session: Represents a single-threaded unit of work and serves as an interface for performing database operations.

Transaction: Provides methods to control and manage database transactions, ensuring data consistency and integrity.

Query: Represents a database query and provides methods to execute and retrieve query results.

Criteria: Provides a programmatic and type-safe way to build and execute database queries using criteria expressions.

Configuration: Represents the configuration settings and properties required for initializing Hibernate, including database connection details and entity mappings.

Q4.What is a Session in Hibernate?
Ans- In Hibernate, a Session represents a single-threaded unit of work that interacts with the database. It provides methods for querying, saving, updating, and deleting persistent objects. The Session acts as a gateway to the underlying database and manages the persistence and retrieval of objects from the database.
-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
Q5.What is a SessionFactory?
Ans- In Hibernate, a SessionFactory is a factory for creating Session instances. It is responsible for initializing Hibernate, configuring database connections, and creating and managing Session objects. The SessionFactory is typically created once during application startup and is thread-safe. It provides a cache of compiled mappings for efficient object-relational mapping operations and serves as a central factory for obtaining Session instances throughout the application.

Q6.What is HQL?
Ans- HQL stands for Hibernate Query Language. It is an object-oriented query language provided by Hibernate for querying and manipulating data in the database using object-oriented concepts. HQL is similar to SQL but operates on persistent objects and their properties rather than database tables and columns. It allows developers to write database queries using the syntax and semantics of their application's programming language (Java) and abstracts away the specific database implementation details.

Q7.What are Many to Many associations?
Ans-Many-to-many associations refer to a relationship between two entities where multiple instances of one entity can be associated with multiple instances of another entity. In other words, it is a bidirectional relationship where both sides can have multiple associations.
-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
Q8.What is hibernate caching?
Ans- Hibernate caching: Hibernate caching is a mechanism that allows Hibernate to store frequently accessed data in memory to improve performance. It reduces the number of database queries by serving data from the cache when it is requested again, thereby reducing the overhead of accessing the database.
-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
Q9.What is the difference between first level cache and second level cache?
Ans- First level cache: Also known as the session cache, it is associated with a Hibernate Session. It is enabled by default and is used to store objects fetched or saved within a session. It provides transaction-level scope and improves performance within a single session.
Second level cache: It is a shared cache that is enabled separately and is used to store objects across multiple sessions. It provides a higher-level scope, typically at the application or cluster level, and helps reduce database access for commonly accessed data.

Q10.What can you tell about Hibernate Configuration File?
Ans-  Hibernate Configuration File: The Hibernate Configuration File (hibernate.cfg.xml) is an XML file used to configure Hibernate. It contains properties such as database connection details, dialect, mapping files, caching settings, and other configuration options required for Hibernate to operate. It is usually placed in the classpath and serves as a central configuration file for the Hibernate framework.
